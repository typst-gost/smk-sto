# smk-sto

Typst-шаблон для оформления учебной отчётности в ФГБОУ ВО
«МГУ им. Н.П. Огарёва». Поддерживаются:

- **отчёты о лабораторных работах** — по **СМК СТО 004–2020**
  «Правила оформления отчётов о лабораторных работах. Общие требования
  к рефератам и аннотациям» (`lab-report` и сопутствующие функции с
  префиксом `lab-report-`);
- **отчёты о практике** — по **СМК СТО 014–2025** «Практика обучающихся
  высшего образования. Общие требования, правила оформления отчётности»
  (`practice-report` и сопутствующие функции с префиксом
  `practice-report-`). Общие требования к оформлению (списка
  использованных источников, приложений, титульного листа и пр.)
  СТО 014–2025 делегирует на **СМК СТО 006–2025** — действующий общий
  стандарт оформления документов учебной деятельности.

Шаблоны закрывают все общие требования стандартов: поля страницы, шрифт,
межстрочный интервал, абзацный отступ, заголовки, перечисления, формулы,
рисунки, таблицы, приложения, и собирают типовые титульные листы
(Приложение А СТО 004 / Приложение В СТО 014).

## Структурные заголовки

«Введение», «Заключение», «Перечень сокращений и обозначений», «Термины
и определения» и подобные разделы по СТО оформляются без номера, по
центру, заглавными буквами. В шаблоне они помечаются меткой `<s>`:

```typst
= Введение <s>
= Заключение <s>
= Перечень сокращений и обозначений <s>
```

Метка — единственный признак, по которому шаблон отличает структурный
заголовок от обычной нумерованной главы. Никакого списка «разрешённых»
названий внутри шаблона нет, поэтому любой свой структурный раздел
достаточно пометить тем же `<s>`. Счётчик глав на таких заголовках не
сдвигается, и в содержании они тоже выводятся без номера и заглавными.

Заголовки, которые Typst сам генерирует без нумерации, — `#outline()`
и `#bibliography(title: ...)` — попадают под то же оформление
автоматически, метка не нужна.

## Блок-схемы и графы

Шаблон переэкспортирует `fletcher` — декларативные блок-схемы прямо в
Typst (узлы и стрелки описываются кодом, без drawio/Visio):

```typst
#figure(
  diagram(
    node((0, 0), [Модуль А],   fill: rgb("#dae8fc")),
    node((2, 0), [Модуль Б],   fill: rgb("#d5e8d4")),
    node((1, 1), [Ядро],       fill: rgb("#ffe6cc")),
    node((3, 1), [Тесты],      fill: rgb("#f8cecc")),
    edge((0, 0), (1, 1), "->", [use]),
    edge((2, 0), (1, 1), "->", [use]),
    edge((3, 1), (1, 1), "->", [test]),
  ),
  caption: [Граф зависимостей модулей],
) <fig:deps>
```

Узлы по умолчанию — скруглённые прямоугольники с тонкой чёрной обводкой
(совместимо с СТО 8.5). Полный API: см. документацию пакета
[fletcher](https://typst.app/universe/package/fletcher).

## Возможности

Общие для обоих стандартов:

- Поля 30/15/20/20 мм, Times New Roman 14 pt, межстрочный интервал 1,5.
- Нумерация разделов «1», «1.1», «1.1.1», абзацный отступ 1,25 см.
- Структурные заголовки — по метке `<s>`: пишется `= Введение <s>`,
  `= Заключение <s>` и т. п., и любой такой `=`-заголовок выводится
  по центру заглавными без номера главы, не сдвигая нумерацию
  остальных разделов. Это работает для любого названия — список
  не хардкодится в шаблоне. Заголовки `#outline()` и
  `#bibliography(title: ...)` уже идут без нумерации, поэтому
  оформляются автоматически.
- Маркер списка «–», нумерация перечислений «1)», поддержка алфавитных
  пунктов «а)», «б)», … (`enum-letter`).
- Подпись таблицы с разрядкой букв «Т а б л и ц а» по СТО 8.6.5.
- Подпись рисунка «Рисунок N – ...» по СТО 8.5.
- Нумерация формул «(1)» справа, ссылки вида «(1)» по СТО 8.4.4.
- Хелпер `where-block(...)` — пояснения к формулам с висячим отступом
  (СТО 8.4.2): «где» начинается с абзаца, переменные выровнены в столбик.
- Приложения через `#show: appendix`: автоматическая буквенная
  нумерация (А, Б, В, …, без Ё, З, Й, О, Ч, Ъ, Ы, Ь по СТО 006–2025
  п. 7.11.4), отдельная перенумерация рисунков, таблиц и формул
  в каждом приложении. Заголовок — «ПРИЛОЖЕНИЕ N» прописными.
  Статус приложения по умолчанию «обязательное»; переопределяется через
  `#show: appendix.with(status: "справочное")` (допустимо «обязательное»,
  «рекомендуемое», «справочное» или `none`).

Лабораторные работы (СТО 004–2020):

- Сборка титульного листа по Приложению А через `lab-report-title-page(...)`.
- Формирование обозначения ЛР: достаточно передать
  `(direction: "27.03.01", variant: "01")` и шаблон соберёт строку
  «ЛР–02069964–27.03.01–01–YY» (`lab-report-designation`).

Отчёт о практике (СТО 014–2025):

- Сборка титульного листа по Приложению В через `practice-report-title-page(...)`.
- Формирование обозначения отчёта:
  `(kind: "У", direction: "27.03.01", variant: "01")` → строка
  «ОП–02069964–У–27.03.01–01–YYYY» (`practice-report-designation`).

## API

Публичные функции с префиксом по стандарту:

| Имя | Назначение |
|---|---|
| `lab-report` | Show-rule отчёта о лабораторной работе (СТО 004–2020). |
| `lab-report-title-page(...)` | Самостоятельный рендер титульного листа (Приложение А). |
| `lab-report-designation(...)` | Форматер «ЛР–02069964–DDD–NN–YY». |
| `practice-report` | Show-rule отчёта о практике (СТО 014–2025). |
| `practice-report-title-page(...)` | Самостоятельный рендер титульного листа (Приложение В). |
| `practice-report-designation(...)` | Форматер «ОП–02069964–В–DD.NN.NN–NN–YYYY». |
| `practice-report-task(...)` | Задание на практику (Приложение Б). |
| `practice-report-diary(...)` | Дневник практики (Приложение Г). |
| `practice-report-survey(...)` | Анкета обучающегося (Приложение Д). |
| `practice-report-feedback(...)` | Отзыв руководителя от профильной организации (Приложение Ж). |
| `appendix` | Show-rule для блока приложений (общая логика). |
| `where-block(...)` | Пояснения к формулам по СТО 8.4.2. |
| `enum-letter(n)` | Алфавитная нумерация «а)», «б)», … |
| `sign-field(position, name)` | Поле подписи: должность — линия — И.О. Фамилия. |
| `table-label(num, caption)` / `table-continuation(num)` | Подписи таблиц. |
| `diagram`, `node`, `edge` | Реэкспорт `fletcher` для блок-схем. |

### Параметры `lab-report`

`lab-report` отвечает только за стилизацию. Все параметры опциональны.

| Параметр         | Тип        | Назначение                                              |
|------------------|------------|---------------------------------------------------------|
| `add-pagebreaks` | `bool`     | Авто-разрывы страниц перед разделами 1-го уровня.       |
| `text-size`      | length     | По умолчанию 14 pt.                                     |
| `small-size`     | length     | Размер мелкого текста (подписи и пр.), по умолчанию 12 pt. |
| `indent`         | length     | Абзацный отступ, по умолчанию 1,25 см.                  |
| `margin`         | dictionary | Поля страницы, по умолчанию 30/15/20/20 мм.             |
| `font`           | строка     | Гарнитура, по умолчанию Times New Roman.                |

### Параметры `lab-report-title-page`

| Параметр        | Тип                | Назначение                                                                   |
|-----------------|--------------------|------------------------------------------------------------------------------|
| `institute`     | строка             | Институт / факультет / филиал.                                               |
| `department`    | строка             | Кафедра.                                                                     |
| `work-number`   | число / строка     | Номер лабораторной работы.                                                   |
| `discipline`    | строка             | Дисциплина.                                                                  |
| `title`         | строка             | Название работы (печатается заглавными).                                     |
| `author`        | словарь            | `(name: "...", direction: "...")`.                                           |
| `supervisor`    | словарь            | `(name: "...", position: "...")`.                                            |
| `designation`   | строка / словарь   | Готовая строка «ЛР–...» либо `(direction: "...", variant: "...")`.           |
| `ministry`      | строка             | По умолчанию — Минобрнауки РФ.                                               |
| `organization`  | словарь            | По умолчанию — МГУ им. Н.П. Огарёва.                                         |
| `city`          | строка             | По умолчанию — «Саранск».                                                    |
| `year`          | число / `auto`     | По умолчанию — текущий год.                                                  |

Поля, не переданные в функцию (значение `none`), не рендерятся — соответствующая строка на титульном листе пропускается целиком.

### Параметры `practice-report`

`practice-report` отвечает только за стилизацию. Принимает те же параметры оформления, что и `lab-report` (`add-pagebreaks`, `text-size`, `small-size`, `indent`, `margin`, `font`).

### Параметры `practice-report-title-page`

| Параметр          | Тип               | Назначение                                                                  |
|-------------------|-------------------|-----------------------------------------------------------------------------|
| `institute`       | строка            | Институт / факультет / филиал.                                              |
| `department`      | строка            | Кафедра.                                                                    |
| `kind`            | строка            | Вид практики: «учебная» / «производственная».                               |
| `practice-type`   | строка            | Тип практики в соответствии с ОПОП ВО (например, «ознакомительная»).        |
| `gender`          | строка            | «male» / «female» / `none` — влияет на словоформы.                         |
| `author`          | словарь           | `(name: "...", course: 1, group: "101М")`.                                  |
| `direction`       | словарь / строка  | `(code: "27.03.01", name: "Стандартизация и метрология")`.                  |
| `profile`         | строка            | Профиль / специализация.                                                    |
| `location`        | строка / content  | Место прохождения практики.                                                 |
| `period`          | словарь / строка  | `(start: "08.09.2025", end: "27.12.2025")`.                                 |
| `designation`     | строка / словарь  | `(kind: "У", direction: "27.03.01", variant: "01")` → «ОП–...».             |
| `supervisor-org`  | словарь           | Руководитель практики от профильной организации (опционально).              |
| `supervisor-uni`  | словарь           | Руководитель практики от Университета.                                      |
| `defense`         | словарь           | `(mark: "5", date: "27.12.2025")` — обычно остаются пустыми на печати.      |
| `ministry`, `organization`, `city`, `year` | …  | Те же значения по умолчанию, что и у `lab-report-title-page`.               |

Поля, не переданные в функцию (значение `none`), не рендерятся — соответствующая строка на титульном листе пропускается целиком.

## Структура отчёта о лабораторной работе по СТО 004

Согласно п. 7.1 отчёт должен содержать:

1. Титульный лист (`#lab-report-title-page(...)`).
2. Цель или задачи работы.
3. Программа и методика эксперимента.
4. Результаты измерений (наблюдений).
5. Обработка экспериментальных данных и оценка погрешностей.
6. Анализ результатов и выводы (заключение).

Опционально: список использованных источников и приложения.

## Структура отчёта о практике по СТО 014

Согласно п. 5.1.1 отчёт состоит из:

1. Титульный лист (`#practice-report-title-page(...)`).
2. Основная часть (содержание проделанной работы в соответствии с
   целями практики и заданием).
3. Заключение (изложение результатов выполнения практики).
4. Список использованных источников (при наличии).
5. Приложения (при наличии).

Дополнительно к отчёту прикладываются (с собственной нумерацией страниц):
задание на практику (`practice-report-task`), дневник практики
(`practice-report-diary`), анкета обучающегося (`practice-report-survey`),
отзыв руководителя от профильной организации (`practice-report-feedback`).

Каждая функция принимает поля, специфичные для формы, и общие сведения
(вид/тип практики, ФИО, курс/группа, направление, профиль). Удобно
вынести общие сведения в словарь и распаковывать через `..`:

```typst
#let common = (
  kind: "учебной",
  practice-type: "ознакомительная",
  gender: "male",
  author: (name: "И. И. Иванов", course: 1, group: "101М"),
  direction: (code: "27.03.01", name: "Стандартизация и метрология"),
  profile: "Метрология и метрологическое обеспечение",
)

#practice-report-task(
  ..common,
  location: "...",
  period: (start: "08.09.2025", end: "27.12.2025"),
  submission-date: "27.12.2025",
  goals: [Формирование навыков работы со средствами измерений.],
  competencies: [УК-1; ОПК-1; ОПК-3],
  tasks: ([Изучить...], [Провести...], [Обработать...]),
  supervisor-uni: (name: "П. П. Петров", position: "канд. техн. наук, доц."),
)

#practice-report-diary(
  ..common,
  entries: (
    ("08.09.2025", [Получение задания.], ""),
    ("09.09.2025 – 27.12.2025", [Выполнение задания.], ""),
  ),
  supervisor-uni: (name: "П. П. Петров", position: "канд. техн. наук, доц."),
)

#practice-report-survey(
  kind: "учебной",
  practice-type: "ознакомительная",
  gender: "male",
  // q1–q11 — поля по списку из Приложения Д.
  answers: (q1: "да", q2: "РПП мне понятна", q7: 5, q10: "полностью удовлетворён"),
)

#practice-report-feedback(
  practice-name: "учебной ознакомительной практики",
  host-org: "ФГБОУ ВО «МГУ им. Н. П. Огарёва», кафедра ...",
  gender: "male",
  author: "И. И. Иванов",
  direction: (code: "27.03.01", name: "Стандартизация и метрология"),
  profile: "...",
  period: (start: "08.09.2025", end: "27.12.2025"),
  self-reliance: [...],
  analysis: [...],
  results: [...],
  attitude: [...],
  professional: [...],
  personal: [...],
  supervisor-org: (name: "П. П. Петров", position: "канд. техн. наук, доц."),
)
```

Каждая форма начинается с новой страницы и не печатает номер страницы
(`set page(footer: [])`) — у Приложений Б, Г, Д, Ж по стандарту своя
нумерация, не входящая в общую нумерацию отчёта.

## Локальная разработка

### Быстрая проверка изменений

```bash
typst compile --root . template/lab-example.typ         # лабораторная
typst compile --root . template/practice-example.typ    # практика
```

Файлы `template/lab-example.typ` и `template/practice-example.typ`
импортируют пакет из `src/lib.typ` напрямую (минуя `@preview`) — это
удобно при разработке: правка → пересборка без копирования файлов.

### Установка в локальный кэш `@preview`

Чтобы протестировать пакет так, как его увидят пользователи после
публикации в [Typst Universe](https://typst.app/universe) — кладём его
в локальный кэш `@preview`. После этого `#import "@preview/smk-sto:0.3.2"`
работает из **любого** проекта на этой машине.

Путь к кэшу зависит от ОС:

| ОС       | Путь                                                              |
|----------|-------------------------------------------------------------------|
| Windows  | `%APPDATA%\typst\packages\preview\smk-sto\0.3.2\`             |
| Linux    | `~/.local/share/typst/packages/preview/smk-sto/0.3.2/`        |
| macOS    | `~/Library/Caches/typst/packages/preview/smk-sto/0.3.2/`      |

Установочные скрипты читают актуальную версию из файла `VERSION` и
копируют пакет в соответствующую папку кэша.

**Windows (PowerShell):**

```powershell
./scripts/install-local.ps1
```

**Linux / macOS:**

```bash
bash scripts/install-local.sh
```

Список исключений в скриптах согласован с полем `exclude` в
[`typst.toml`](typst.toml).

### Проверка из стороннего проекта

После установки в кэш:

```bash
mkdir /tmp/check && cd /tmp/check
echo '#import "@preview/smk-sto:0.3.2": *
#show: lab-report
#lab-report-title-page(work-number: 1, title: "Проверка",
  designation: (direction: "27.03.01", variant: "01"))
= Раздел
Текст.' > report.typ
typst compile report.typ
```

Если PDF собрался — пакет корректно установлен и готов к использованию.
Тем же путём (`typst init @preview/smk-sto my-report`) можно
создать новый проект из вшитого в пакет шаблона.

### Обновление пакета в кэше

После любых правок исходников просто повторите команду установки. Кэш
полностью заменяется — никакой ручной чистки не нужно.
